# 面试策略

## 简历项目包装

### 怎么写 K8s 项目经历？

面试官看简历时最关心：**你做了什么、多大规模、遇到什么问题、怎么解决的。**

#### 模板（STAR 法则）

```text
项目名：XX 公司容器化平台
角色：K8s 运维工程师 / SRE

背景：公司 200+ 微服务从 VM 迁移到 K8s
任务：负责集群搭建、CI/CD 流水线、监控告警体系
行动：
  - 搭建 3 节点生产集群（Calico CNI + IPVS）
  - 设计 GitOps 流水线（ArgoCD + GitHub Actions）
  - 部署 Prometheus + Grafana 监控体系
结果：
  - 部署频率从每周 1 次提升到每天 10+ 次
  - 故障恢复时间从 30 分钟缩短到 5 分钟
```

#### 常见错误

| 错误写法 | 改进写法 |
|----------|----------|
| "负责 K8s 集群运维" | "搭建 50 节点 K8s 集群，管理 200+ 微服务" |
| "使用 Helm 部署应用" | "编写 Helm Chart 模板化 30+ 应用部署，统一 values 分层管理" |
| "做过扩缩容" | "实现 HPA 自动扩缩容，峰值流量时自动从 3 扩到 15 Pod" |

### 没有真实 K8s 项目怎么办？

用本指南的实验构建项目经历：

1. 在本地用 Kind 搭建集群
2. 部署一个多微服务应用（前端 + 后端 + 数据库）
3. 配置 CI/CD（GitHub Actions → 镜像构建 → ArgoCD 部署）
4. 配置监控（Prometheus + Grafana）
5. 写成项目文档，面试时能讲清楚每一步

## 面试流程解析

### K8s 岗位典型面试流程

```text
第一轮：技术基础（30-45 分钟）
├── K8s 核心概念（Pod/Deployment/Service）
├── Linux 基础（网络、进程、文件系统）
└── Docker 基础（镜像、容器、网络）

第二轮：深入技术（45-60 分钟）
├── 你的专长领域深挖（如网络/调度/存储）
├── 场景设计题（设计一个高可用方案）
└── 故障排查（给一个场景让你分析）

第三轮：系统设计（60 分钟）
├── 设计一个容器平台
├── 设计 CI/CD 流水线
└── 设计监控告警体系

第四轮：行为面试（30 分钟）
├── 项目管理经验
├── 团队协作
└── 职业规划
```

### 各轮准备重点

| 轮次 | 重点准备 | 参考资料 |
|------|----------|----------|
| 基础轮 | 核心概念 + 命令操作 | 初学者轨道 |
| 深入轮 | 你的深潜领域 | 领域深潜 + 面试题 |
| 设计轮 | 架构思维 +  trade-off | 系统全景图 |
| 行为轮 | STAR 法则 + 项目经历 | 本文简历部分 |

## K8s 岗位需求分析

### 常见 K8s 岗位

| 岗位 | K8s 深度要求 | 核心技能 |
|------|-------------|----------|
| **K8s 运维工程师** | 深 | 集群搭建、故障排查、监控告警 |
| **SRE** | 深 | 可靠性设计、SLI/SLO、自动化运维 |
| **DevOps 工程师** | 中 | CI/CD、GitOps、Helm |
| **后端开发（云原生方向）** | 中 | Service Mesh、gRPC、微服务设计 |
| **平台工程师** | 深 | 多租户、RBAC、自定义 Controller |

### 面试高频话题分布

基于公开招聘 JD 和面试经验分享统计：

| 话题 | 出现频率 | 准备优先级 |
|------|----------|-----------|
| Pod/Deployment/Service 基础 | 极高 | ⭐⭐⭐ |

| 监控（Prometheus/Grafana） | 中 | ⭐⭐ |
| 故障排查 | 高 | ⭐⭐⭐ |
| 场景设计题 | 高 | ⭐⭐⭐ |

### 面试中的加分项

1. **有 CKA/CKAD 认证**：证明系统性知识
2. **能说清 trade-off**：不是"XX 最好"，而是"在 XX 场景下选 XX 因为 XX"
3. **有源码级理解**：能聊 kubelet 的 CRI 调用链、scheduler 的 filter/score 流程
4. **有生产经验**：处理过线上故障、做过容量规划
5. **关注社区动态**：了解 Gateway API、Sidecar Container（KEP-753）等新方向
