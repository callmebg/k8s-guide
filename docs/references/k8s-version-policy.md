# Kubernetes 版本标注策略

## 概述

K8s Guide 中的所有文章均标注了验证时使用的 Kubernetes 版本。本文档说明版本标注的含义、维护规则和兼容性承诺。

## 版本含义

每篇文章的 frontmatter 中包含 `k8sVersion` 字段：

```yaml
---
k8sVersion: "v1.31.0"
---
```

此字段表示：**本文的命令、配置和概念说明在 Kubernetes v1.31.0 环境下经过验证，可以正常工作**。

### 这不是兼容性保证

`k8sVersion` 是**经验验证版本**，不是兼容性范围声明：

- ✅ 文章内容在标注版本上经过实操验证
- ⚠️ 文章内容在标注版本的相邻小版本上**大概率**兼容（如 v1.31.0 标注的内容通常适用于 v1.30.x 和 v1.32.x）
- ❌ 文章不保证在标注版本的大版本差距下（如 v1.20 vs v1.31）完全兼容

## Kubernetes 版本节奏

Kubernetes 每年发布 3 个小版本（约每 4 个月一次）：

```
v1.31 → v1.32 → v1.33 → v1.34 → ...
```

每个小版本有约 14 个月的补丁支持窗口。K8s Guide 的维护目标是：

| 差距 | 策略 | 说明 |
|------|------|------|
| **0-1 个小版本落后** | 🟢 理想状态 | 内容保持新鲜，定期跟进 |
| **2-3 个小版本落后** | 🟡 需要关注 | 部分 API 可能已弃用，标注警告 |
| **4+ 个小版本落后** | 🔴 需要更新 | 大量 API 可能已移除，优先修复 |

## 维护规则

### 对于贡献者

1. **新增文章**：在 frontmatter 中填写你验证时使用的 K8s 版本
2. **更新文章**：当你重新验证内容在新版本上可用时，更新 `k8sVersion` 字段
3. **发现不兼容**：如果发现某篇文章在较新版本上不工作，请提交 Issue 或 PR

### 对于读者

1. **版本接近**：如果文章标注版本与你使用的版本相差 ≤ 2 个小版本，内容大概率适用
2. **版本差距大**：如果相差 ≥ 3 个小版本，建议：
   - 优先关注概念和原理（变化小）
   - 对命令和 YAML 配置保持警惕（可能已变更）
   - 参考 [Kubernetes 官方文档](https://kubernetes.io/docs/) 确认 API 变更

## 当前状态

| 项目 | 版本 |
|------|------|
| **目标 K8s 版本** | v1.31.0 |
| **内容最后全量验证** | 2026-07 |
| **下次计划更新** | K8s v1.34.0 发布后 |

## 版本更新流程

当需要更新到新 K8s 版本时：

1. 创建追踪 Issue（如 `chore: re-verify content against K8s v1.34`）
2. 逐篇验证关键命令和 API 是否仍然可用
3. 更新文章的 `k8sVersion` frontmatter
4. 更新本文档的版本状态表
5. 在 CHANGELOG 中记录
